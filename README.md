# FallingUp

## 概要
壁をタップすることで重力の向きを変更し、ゴールを目指す3Dパズルゲームです。

## 🎮 プレイリンク
**[itch.ioでプレイする](https://4414.itch.io/fallingup)**

## ゲームの特徴
- 壁をクリックすることで、重力方向を自由に変更できるシステム
- ステージは3つ作成済み
- Stage 3では、部屋から部屋へ重力を使って「落下」しながら移動する新しい体験

## 操作方法
- **マウスクリック**：壁をクリックすると、その方向に重力が変化
- **WASDキー**：プレイヤー移動
- **マウス移動**：視点移動


## ステージ構成
### Stage 1（チュートリアル）
重力操作の基本を学ぶ、シンプルな構造。
開始と同時に目の前の壁には「WASDで移動すること」と「壁をタップすること」がわかるようテキストを配置
壁に一目で分かるゴールを設置

### Stage 2
開始時点の視点ではどこにゴールがあるか分からない
重力の操作や移動でゴールを見つける必要がある

### Stage 3（2部屋構造）
部屋から部屋へ、重力を使って「落下」しながら移動する。
「重力を操作して進める」というコアメカニクスを活かした、パズル体験。
ポートフォリオとして、重力操作というコアメカニクスが伝わる最小構成を意識し、3ステージ構成としました。

## 技術的な工夫・学んだこと

### 重力制御システム
壁をクリックした際、RaycastでヒットしたColliderの法線ベクトルを取得し、`Physics.gravity`を動的に変更することで重力方向を切り替えています。

### UI・3D空間の判定分離
`EventSystem.current.IsPointerOverGameObject()`を使用し、UIクリック時は3D空間のRaycastをスキップすることで、ボタンと壁判定の干渉を回避しました。

### カーソルロック制御
FPS風の操作感のため通常はカーソルをロック（`Cursor.lockState = CursorLockMode.Locked`）していますが、ゴール時には`CursorLockMode.None`に変更してUIを操作可能にしています。

### 物理演算の最適化
プレイヤーのRigidbodyに`freezeRotation = true`を適用し、重力方向が変わっても転倒しないよう制御しています。

### レベルデザイン
Stage 3では、1つの部屋構造から2つの部屋 + 通路という構造に変更し、「重力で落ちて次の部屋へ移動する」というパズル体験を実現しました。コアメカニクスを活かしながら、段階的に難易度を上げる設計を意識しています。
